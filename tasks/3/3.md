# Задание 3

Реализовать валидацию запроса создания контакта.

## Функциональные требования

1. Сервис проверяет все поля запроса на добавление нового контакта согласно
   `OpenAPI` схеме, приведенной ниже:

   ```yaml
   components:
     schemas:
       Contact:
         type: object
         required:
           - first_name
           - last_name
           - birthday
         properties:
           id:
             type: integer
             format: int64
             readOnly: true
           first_name:
             type: string
             minLength: 1
             maxLength: 255
           middle_name:
             type: string
             maxLength: 255
           last_name:
             type: string
             minLength: 1
             maxLength: 255
           birthday:
             type: string
             format: date
           description:
             type: string
             maxLength: 255
   ```

1. Сервис должен возвращать ошибки с кодом `400` в случае невалидного запроса на
   создание контакта.

1. Возвращаемая ошибка должна быть в формате `JSON` и содержать причину.
   Допустимые значения для объекта `Contact` указаны в `OpenAPI` схеме выше.

   ```yaml
   components:
     schemas:
       ValidationError:
         type: object
         required:
           - field
           - reason
         properties:
           field:
             type: string
           reason:
             type: string
       Errors:
         type: object
         required:
           - errors
         properties:
           errors:
             type: array
             items:
               $ref: '#/components/schemas/ValidationError'
   ```

## Нефункциональные требования

1. Для валидации запроса можно использовать любую удобную библиотеку или
   написать свою.

1. Поле `birthday` должно проверяться на соответствие формату `full-date`
   согласно [RFC 3339](https://www.rfc-editor.org/rfc/rfc3339#section-5.6)
   (пример: `1799-06-06`).
